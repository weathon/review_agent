# Harsh Critic Review
## Section-by-Section Critical Review

### Title & Abstract
- **Does the title accurately reflect the contribution?** Yes. The title clearly signals a dynamic, adaptive measurement framework for LLM value differences, which aligns with the paper's core algorithmic and empirical contribution.
- **Does the abstract clearly state the problem, method, and key results?** Yes. It concisely defines the "informativeness challenge" (static benchmarks yield saturated results), outlines the method (iterative, in-context optimization of an information-theoretic objective using LLM value boundaries), and summarizes the outcomes (generation of a novel, extensible benchmark with validation of its effectiveness).
- **Are any claims in the abstract unsupported by the paper?** The claim that AdAEM "theoretically maximizes an information-theoretic objective" is slightly overstated. As revealed in Appendix C.3 and Section I, the practical implementation relies on heuristic approximations (e.g., Jaccard similarity, BERTScore, binary classifier unions) rather than exact probability maximization. While common in black-box LLM optimization, this weakens the strict theoretical maximization claim. Additionally, "co-evolve with the development of LLMs" is conceptually sound but remains aspirational without longitudinal experiments tracking benchmark updates over new model releases.

### Introduction & Motivation
- **Is the problem well-motivated? Is the gap in prior work clearly identified?** Yes. The introduction effectively motivates the "informativeness challenge," demonstrating that static value benchmarks suffer from data contamination and alignment-induced saturation, leading to indistinguishable model profiles (supported by Fig. 1). The gap for a dynamic, self-extensible value evaluation framework is clearly articulated.
- **Are the contributions clearly stated and accurate?** Yes. The three contributions are explicit: (1) proposing the first self-extensible dynamic value evaluation algorithm, (2) demonstrating automated generation of diverse/value-evoking questions, and (3) releasing AdAEM Bench with comprehensive analysis. These align closely with the paper's content.
- **Does the introduction over-claim or under-sell?** The claim "we are the first to propose a novel self-extensible dynamic value evaluation method" should be slightly nuanced. While likely true for *value-specific* benchmarks, recent dynamic evaluation frameworks for reasoning/safety (e.g., DyVal, Benchmark Self-Evolving) share similar architectural philosophies. Qualifying the claim to "first dynamic evaluation framework specifically targeting latent value orientations" would be more precise and avoid reviewer pushback.

### Method / Approach
- **Is the method clearly described and reproducible?** The high-level pipeline (Sec 3.2, Fig 2, Alg 1) is clear, but reproducibility is hampered by the heavy reliance on the appendix for core implementation details. Crucially, the derivation in the main text implies a continuous probabilistic optimization, while Appendix C.3 replaces terms with discrete heuristics. A clearer bridge between theory and practice in the main text would improve reproducibility.
- **Are key assumptions stated and justified?** The assumption that LLM values are latent variables best elicited through controversial social prompts is stated and reasonably justified via prior psych/social science literature and the authors' own validation. However, the assumption that LLM-generated responses can reliably approximate the true conditional distributions $p_\theta(y|v)$ and $p_\theta(v|y)$ via black-box classifiers is non-trivial and not fully justified.
- **Are there logical gaps in the derivation or reasoning?** Yes. Eq. 2 and Eq. 3 are derived from variational bounds on KL/JS divergence, but the practical scoring function in Appendix C.3 substitutes probability terms with heuristic metrics (e.g., Jaccard for value diversity, BERTScore for opinion diversity). There is no theoretical justification provided for why these heuristics tightly bound or approximate the original information-theoretic objective. This creates a gap between the claimed IM/EM framework and the actual optimization landscape.
- **Are there edge cases or failure modes not discussed?** The framework could theoretically optimize toward adversarial or semantically degenerate prompts that artificially inflate heuristic scores without eliciting genuine value reflection. While E.5 shows examples of low/high scores, there is no discussion of convergence guarantees or mechanisms to prevent reward hacking/over-optimization as $B$ increases.
- **For theoretical claims: are proofs correct and complete?** The derivation in Appendix D follows standard Information Maximization principles and is mathematically sound in its formalism. However, the approximations noted in Appendix I (Total Variation lower bounds for JS, Monte Carlo sampling for expectations, binary classifier approximations) transform the proof into a variational lower-bound optimization rather than a strict maximization. The authors acknowledge this, but the main text should temper the theoretical claims accordingly.

### Experiments & Results
- **Do the experiments actually test the paper's claims?** Yes. The experiments test distinguishability (Fig 8), novelty/diversity (Table 1, Fig 3), validity (priming experiments, Sec 4.2), and extensibility (regional/temporal analysis, Sec 4.3). These directly support the stated contributions.
- **Are baselines appropriate and fairly compared?** SVS, ValueBench, and ValueDCG are appropriate static baselines for value evaluation. The comparison is fair, but lacks statistical significance testing (e.g., permutation tests, confidence intervals) to confirm that the observed improvements in "distinguishability" over baselines are robust beyond a single run.
- **Are there missing ablations that would materially change conclusions?** Yes. Critical ablations are missing: (1) Component ablation of the heuristic score $S(x)$ in Eq. 3 (e.g., removing "opinion diversity" or "context coherence" terms to show each term's contribution to final benchmark quality); (2) Ablation on the number of LLMs in P1/P2 sets to justify the computational design; (3) Comparison against a non-dynamic but randomly expanded question set to isolate the benefit of the optimization vs. mere scale.
- **Are error bars / statistical significance reported?** Partially. The value priming experiments report p-values and Cohen's d, which is good. However, the main benchmark generation and evaluation results lack run-to-run variance metrics (e.g., standard deviation across multiple AdAEM seeds), which is important given the stochastic nature of LLM generation and MAB exploration.
- **Do the results support the claims made, or are they cherry-picked?** Results generally support the claims, but Fig 8 and associated discussions focus heavily on favorable visual separations without quantitative metrics of cluster separation or inter-model variance. The claim that AdAEM Bench is "highly informative" is supported but would be stronger with inter-rater reliability or human agreement scores on the *evaluation outputs*, not just the questions.
- **Are datasets and evaluation metrics appropriate?** Schwartz's value theory is a standard, appropriate choice. The use of TrueSkill for relative ranking is clever for handling uncertainty. However, the primary value labeler is GPT-4o (App C.2). This introduces potential circularity/classifier bias: if GPT-4o systematically favors certain linguistic patterns or aligned responses, it may artificially inflate/deflate value scores for other models. The TrueSkill mitigation is noted but not empirically isolated.

### Writing & Clarity
- **Are there sections that are confusing or poorly explained?** The transition between the theoretical derivation (Sec 3.2, Eq. 2-3) and the practical implementation (Appendix C.3) is confusing. Readers expecting a probabilistic optimization are met with discrete heuristic substitutions without clear motivation in the main text. Additionally, the notation switches between $p_\theta(v|x)$ and $p^x_i(v)$, and the distinction between the optimization phase (using P1) and evaluation phase (using P2) could be more explicitly separated in the methodology to avoid confusion about data contamination.
- **Are figures and tables clear and informative?** Mostly yes. Table 1 clearly shows benchmark statistics. Fig 8 effectively visualizes model distinguishability across benchmarks. However, Fig 3 (t-SNE visualization) lacks crucial metadata: which embedding model was used for the questions? t-SNE is highly sensitive to hyperparameters and embeddings, so without this information, the visual claim of "better semantic diversity" is difficult to independently assess.

### Limitations & Broader Impact
- **Do the authors acknowledge the key limitations?** Yes. They acknowledge the use of Schwartz's theory (not exhaustive), English-centric scope, computational constraints limiting P1/P2 pools, and potential misuse of controversial prompts.
- **Are there fundamental limitations they missed?** (1) **Classifier Bias:** As noted, relying on GPT-4o to label values introduces systematic measurement bias that could skew all comparative results. (2) **Disentanglement Efficacy:** The disentanglement term aims to separate question bias from model values, but there is no empirical validation showing this term actually reduces question-induced value inflation compared to a version without it. (3) **Cultural Representation in P1/P2:** The method's ability to capture cross-cultural values heavily depends on the geographic and alignment diversity of P1/P2. If future evaluations use a less diverse pool, the "co-evolution" and boundary-probing capabilities degrade significantly.
- **Are there failure modes or negative societal impacts not discussed?** The intentional generation of highly controversial/polarizing questions could be weaponized for large-scale value-targeted red-teaming or generating divisive content, beyond the general "malicious use" warning. The paper should more directly address safeguards for releasing a benchmark of 12k intentionally divisive prompts, even if filtered by Llama-Guard, as value controversies are highly context-dependent and guardrails are imperfect.

### Overall Assessment
This paper tackles a highly relevant and well-motivated problem in LLM alignment: moving beyond static, saturated value benchmarks toward a dynamic, self-extensible evaluation paradigm. The mathematical framing via information maximization is elegant, and the empirical results convincingly demonstrate that AdAEM generates more diverse, culturally/temporally relevant questions that better distinguish model value profiles than existing baselines. The validation via controlled value priming and the TrueSkill aggregation metric are strong points that align well with ICLR's emphasis on rigorous evaluation methodologies.

However, significant concerns must be addressed before full acceptance. First, there is a notable disconnect between the theoretical information-theoretic optimization presented in the main text and the heuristic scoring functions detailed in the appendix. The authors should either tighten the theoretical justification for these heuristics or reframe the main text to accurately reflect the practical implementation. Second, the evaluation lacks critical ablations on the objective function's components and statistical significance testing for benchmark comparisons. Third, the heavy reliance on an LLM-as-a-judge (GPT-4o) for value labeling introduces unquantified classifier bias that threatens the validity of the reported value differences. If the authors can bridge the theory-practice gap, provide component ablations, and thoroughly analyze/quantify the impact of the value classifier's biases, this work would represent a substantial and timely contribution to dynamic LLM evaluation at ICLR.

# Neutral Reviewer
## Balanced Review

### Summary
This paper introduces AdAEM, a dynamic and self-extensible evaluation framework designed to address the "informativeness challenge" in LLM value measurement, where static benchmarks yield saturated, indistinguishable results due to data contamination and generic prompting. By formulating question generation as an information-theoretic optimization problem combined with a Multi-Armed Bandit exploration strategy, AdAEM adaptively probes the value boundaries of diverse LLMs to produce highly controversial and culturally/temporally distinct test questions. Comprehensive evaluations across Schwartz's theory and Moral Foundations Theory demonstrate that AdAEM generates high-quality, novel questions that reliably elicit distinguishable value differences across a wide spectrum of models.

### Strengths
1. **Addresses a Critical Evaluation Bottleneck:** The paper clearly identifies and tackles benchmark saturation and data contamination in value alignment research. By shifting from static curation to an adaptive, self-extensible pipeline, it offers a principled solution to a recognized community pain point (Sec. 1, Fig. 1).
2. **Strong Theoretical Grounding:** The formulation of the informativeness objective using Generalized Jensen-Shannon divergence and KL regularization (Eq. 1) provides a clear mathematical foundation for maximizing model distinguishability while minimizing prompt-induced bias. The EM-like alternating optimization framework (Sec. 3.2, App. D) is theoretically sound and well-motivated.
3. **Comprehensive Empirical Validation:** The authors provide rigorous validation across multiple axes: human evaluation shows +52% improvements in controversy and value elicitation (App. C.10); construct validity is confirmed via controlled value priming with strong effect sizes (Sec. 4.2, Tab. 10, Tab. 12); and reliability is demonstrated via high Cronbach’s $\alpha$ (~0.90) across random dataset folds (Sec. 4.2, App. K.3).
4. **Demonstrated Extensibility and Cultural/Temporal Sensitivity:** The framework successfully captures regional biases and temporal knowledge cutoffs by leveraging diverse LLM pools (Fig. 5, Fig. 6), proving its capability to co-evolve with model development. Instantiation on a secondary value system (Moral Foundations Theory, Sec. J) further supports generalizability.

### Weaknesses
1. **Theory-Implementation Gap:** While the theoretical derivation in Sec. 3.2 is elegant, the practical implementation heavily relies on heuristic approximations (BERTScore for semantic difference, Jaccard for value diversity, LLM classifiers for value conformity) due to black-box constraints. Although Appendix I shows a ~0.85 correlation with exact probabilities, the disconnect between the formal objectives (Eq. 2-3) and the implemented scoring functions (App. C.3) weakens the mathematical rigor expected at ICLR.
2. **Potential Circularity/Overfitting to Generator Pool:** Questions are optimized to maximize value differences specifically across the P1/P2 model pools. There is no explicit ablation testing whether questions generalize to models *completely held out* from the generation pipeline. Optimized questions might inadvertently reflect the latent value priors or failure modes of the specific LLMs used in P1/P2 rather than universal value boundaries.
3. **High Computational Overhead for Practical Deployment:** The iterative EM refinement combined with MAB exploration requires extensive API calls to multiple frontier models. While the authors frame this as a strength (self-extensibility), the real-world cost and latency (budget $B=1500$, multiple rounds per topic) are not broken down or compared against static benchmarks, raising questions about practical accessibility for researchers with limited compute.
4. **Exposition and Notation Density:** Section 3.2 introduces heavy notation and probabilistic terms without sufficient high-level intuition. The transition from the mathematical formulation to Algorithm 1/2 is abrupt, and Figure 2 lacks the granularity needed to clarify how the MAB arm selection interacts with the EM score maximization in practice.

### Novelty & Significance
**Novelty:** High. The paper presents a novel intersection of dynamic benchmark generation, information-theoretic optimization, and AI value evaluation. Framing question elicitation as an EM-style optimization to probe model "value boundaries" is a fresh conceptual contribution that moves beyond template-based or purely LLM-as-judge approaches.
**Significance:** High, with practical caveats. If adopted, AdAEM could substantially improve the reliability of AI alignment and cultural bias assessments by mitigating memorization and benchmark saturation. It offers a methodological shift toward co-evolving evaluation pipelines. For ICLR, the work is significant as it bridges evaluation methodology, information theory, and alignment research. However, its broader impact hinges on community trust in the approximation methods and the feasibility of its computational requirements.

### Suggestions for Improvement
1. **Explicitly Map Heuristics to Theoretical Terms:** In Sec. 3.2 or App. C.3, add a clear table or paragraph mapping each theoretical term in Eq. (2) and Eq. (3) to its practical approximation. Discuss the error bounds or empirical justification for each substitution to strengthen the theory-practice bridge.
2. **Ablate Generalizability Beyond P1/P2:** Conduct an experiment where AdAEM generates questions using Pool A (P1/P2), but evaluation is performed on a completely disjoint Pool B (e.g., newly released or architecturally distinct models). Report distinguishability and reliability metrics to prove the questions capture universal value tensions rather than fitting the generator pool's distribution.
3. **Provide Compute/Cost Breakdown and Efficiency Strategies:** Include a detailed table quantifying API calls, latency, and estimated cost per benchmark generation cycle. Discuss potential optimizations (e.g., distilling the scoring function into a smaller reward model, caching similar queries, or asynchronous MAB exploration) to lower the barrier for community adoption.
4. **Improve Methodological Flow and Visualization:** Add a unified architectural diagram that explicitly shows the data flow between the MAB topic selector, the EM optimizer, and the TrueSkill evaluator. Simplify notation in Sec. 3.2 by introducing a glossary upfront and using consistent subscripts for models, questions, and responses. Clarify how disentanglement is practically enforced without fine-tuning.

# Spark Finder Review
## How to Improve This Paper

### Missing Experiments (top 3-5 only)
1. Ablate the four terms in the informativeness objective (value diversity, opinion diversity, semantic coherence, disentanglement). Without it, the claim that the full information-theoretic formulation drives results is unverified, as one proxy metric may dominate the optimization.
2. Compare against dynamic or evolutionary evaluation baselines (e.g., S-Eval, LatestEval, or prompt-optimization methods like PromptAgent). Benchmarking only against static questionnaires fails to prove AdAEM’s superiority within the dynamic evaluation paradigm it claims to advance.
3. Empirically verify contamination resistance by testing if models fine-tuned or heavily prompted on AdAEM questions quickly saturate in value scores. The claim that generative extension inherently mitigates data leakage remains purely theoretical without this stress test.

### Deeper Analysis Needed (top 3-5 only)
1. Analyze the construct validity of the value classifier by identifying which linguistic features or surface triggers actually drive high value scores versus deep value reasoning. Without this, high "distinguishability" may simply reflect prompt sensitivity or lexical variance rather than genuine value alignment differences.
2. Quantify the empirical impact of the disentanglement term in decoupling question bias from model output. The paper provides a mathematical lower bound but lacks quantitative evidence (e.g., measuring score shifts when question framing is perturbed while holding model identity constant).
3. Report confidence intervals or variance metrics around the TrueSkill-aggregated value rankings across different random seeds or question subsamples. Making strong comparative claims about model values without statistical uncertainty bounds is methodologically fragile.

### Visualizations & Case Studies
1. Provide a circumplex visualization of the 10 Schwartz dimensions derived from AdAEM scores to verify alignment with the established psychological value structure. This would prove the method captures meaningful human value geometry rather than optimization artifacts driven by semantic diversity alone.
2. Show side-by-side qualitative examples of how high-scoring questions force divergent stances across models versus low-scoring questions that trigger generic refusals. This would directly demonstrate the "controversy elicitation" mechanism central to the paper's motivation.
3. Include a model-vs-model divergence heatmap showing informativeness scores before and after AdAEM optimization. A clear visual contrast would concretely prove the "indistinguishable to distinguishable" transition claimed in the introduction.

### Obvious Next Steps
1. Conduct an expert human evaluation on the final model value rankings to verify that AdAEM’s relative distinguishability aligns with established sociological expectations of the evaluated models' training data and cultural origins. Without this, informativeness is only mathematically, not practically, validated.
2. Provide a rigorous compute-to-performance tradeoff analysis (API costs, iteration counts, wall-clock time) against baseline benchmarks. ICLR reviewers require clear evidence that iterative multi-LLM optimization pipelines are practically feasible rather than prohibitively expensive.
3. Expand question generation and evaluation to non-English languages to substantiate the "cultural adaptability" claim. Generating culturally diverse topics in English only from Western/Chinese corporate models does not robustly prove cross-cultural value discovery.

# Final Consolidated Review
## Summary
This paper introduces AdAEM, a dynamic and self-extensible framework for measuring value differences in LLMs. To address the "informativeness challenge" where static benchmarks yield saturated, indistinguishable results due to data contamination and over-alignment, AdAEM formulates question generation as an information-theoretic optimization problem. It iteratively refines prompts using an EM-style algorithm and explores topics via a Multi-Armed Bandit, probing the value boundaries of diverse models. The resulting benchmark demonstrates high semantic diversity, strong construct validity via value priming experiments, and the ability to reveal culturally and temporally distinct value orientations across 16 LLMs.

## Strengths
- **Novel Dynamic Formulation for a Saturated Field:** Shifts value evaluation from static curation to an adaptive, information-maximization framework that explicitly targets benchmark saturation. By optimizing for model distinguishability while enforcing disentanglement from question bias, it directly solves the identified "informativeness challenge."
- **Rigorous Multi-Axis Validation:** Provides strong empirical grounding through controlled value priming (demonstrating large effect sizes, e.g., Cohen’s d > 1.2, for construct validity), human evaluation (+52.8% improvement in controversy/value elicitation), and high reliability (Cronbach’s α ≈ 0.90 across random dataset partitions). The results convincingly show that generated questions are both high-quality and reliably elicit latent value differences.
- **Demonstrated Extensibility and Theoretical Generalizability:** The framework successfully operationalizes "co-evolution" by leveraging diverse model pools to capture temporal knowledge cutoffs and regional biases in generated questions. Instantiation on a secondary value system (Moral Foundations Theory) with consistent results further validates the method's adaptability beyond Schwartz's theory.

## Weaknesses
- **Missing Component Ablations for the Optimization Objective:** The informativeness score $S(x)$ combines multiple theoretical terms (value diversity, opinion diversity, semantic coherence, and disentanglement). The paper does not ablate these components to verify that each contributes meaningfully to the final benchmark quality, or if a single heuristic dominates the optimization. Without this, the claim that the full information-theoretic formulation drives the results remains partially unverified empirically.
- **Unquantified Systematic Bias from the LLM Value Classifier:** The framework relies heavily on GPT-4o/Mini to extract value dimensions from model responses. While classifier accuracy is reported (~92%), unexamined systematic biases in the judge’s linguistic preferences or alignment priors could skew the relative rankings. TrueSkill aggregation mitigates absolute saturation but does not fully isolate whether observed model differences stem from genuine value divergence or the classifier’s consistent preference for specific rhetorical styles or refusal patterns.

## Nice-to-Haves
- **Explicit Comparison to a Randomly Expanded Baseline:** A head-to-head evaluation against a randomly expanded set of controversial questions (without the EM optimization) would better isolate the specific value added by the information-theoretic refinement versus sheer scale/diversity.
- **Compute/Cost Breakdown for Benchmark Generation:** Providing a detailed breakdown of API calls, wall-clock time, and estimated cost per generation cycle would help the community assess the practical feasibility and accessibility of deploying this pipeline for routine evaluation.
- **Main Text Theory-Practice Bridge:** While Appendix I empirically validates the correlation (0.85 Pearson) between the exact mathematical formulation and the heuristic implementation, adding a clear mapping table or paragraph in Section 3.2 would significantly improve readability and reproducibility without requiring readers to navigate the appendix.

## Novel Insights
The paper’s core insight—that LLM value differences can be systematically elicited by treating benchmark construction as an adversarial, information-maximizing game between a question generator and a diverse pool of models—reframes dynamic evaluation from a mere anti-contamination tactic to a principled discovery process. By formalizing "disentanglement" as a regularization term against prompt bias and leveraging relative ranking (TrueSkill) to bypass saturation, the work provides a robust methodological template for measuring other latent, multidimensional LLM traits where static benchmarks inevitably flatten model distinctions.

## Suggestions
- Conduct a targeted ablation study by zero-weighting or removing each component of the scoring function $S(x)$ during generation. Report downstream impacts on human evaluation scores and final model distinguishability to verify the necessity and marginal contribution of each term.
- Perform a sensitivity analysis on the value classifier by cross-evaluating a subset of model responses with an alternative judge (e.g., a fine-tuned value classifier or a different frontier model) and reporting the correlation in TrueSkill rankings. This would directly address potential circularity and strengthen confidence in the reported value differences.
- Add a unified architectural diagram or clear mapping in the methodology section that explicitly shows how the MAB topic selector, the EM optimizer, and the TrueSkill evaluator interact in practice, simplifying notation for faster reader comprehension.
