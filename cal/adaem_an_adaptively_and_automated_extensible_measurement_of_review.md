# Harsh Critic Review
## Section-by-Section Critical Review

### Title & Abstract
- **Does the title accurately reflect the contribution?** Yes. It precisely captures the core idea: an adaptive, automated, and extensible framework for measuring LLM value differences.
- **Does the abstract clearly state the problem, method, and key results?** The abstract clearly defines the "informativeness challenge" (static benchmarks yield saturated/indistinguishable results), introduces AdAEM as a self-extensible algorithm using information-theoretic optimization and in-context probing, and states the key outcome: generating novel, distinguishable value questions that co-evolve with LLMs.
- **Are any claims in the abstract unsupported by the paper?** The phrase "theoretically maximizes an information-theoretic objective" slightly overstates the practical implementation, as the actual optimization relies on heuristic proxies (detailed in the Method section). Additionally, "co-evolve with the development of LLMs" is conceptually sound but operationally means the pipeline can be re-run with newer models; the abstract slightly oversells this as an inherent property rather than a deployment choice.

### Introduction & Motivation
- **Is the problem well-motivated? Is the gap in prior work clearly identified?** Yes. The saturation of LLM evaluations on post-trained safety dimensions (e.g., HHH) and data contamination in static benchmarks are well-documented. The gap—lack of dynamic, self-extending evaluation paradigms specifically for *value differences*—is clearly articulated and grounded in measurement theory.
- **Are the contributions clearly stated and accurate?** The three contributions are well-scoped: (1) proposing the dynamic self-extensible method, (2) demonstrating generation quality/value differentiation, and (3) releasing the benchmark with validation. They align closely with the paper's content.
- **Does the introduction over-claim or under-sell?** It appropriately positions itself within dynamic evaluation literature. The claim to be the "first self-extensible dynamic value evaluation method" is plausible, though the introduction could better acknowledge related dynamic QA/safety benchmarks (e.g., S-Eval, LatestEval) to sharply delineate the *value-specific* novelty.

### Method / Approach
- **Is the method clearly described and reproducible?** The high-level framework (MAB for exploration, EM-like alternation for optimization) is clearly presented. However, full reproducibility depends heavily on prompt templates and heuristic scoring functions only detailed in Appendix C.3, which are absent from the main methodology.
- **Are key assumptions stated and justified?** Key assumptions include: (1) LLM responses reliably reflect latent value orientations, (2) an LLM-as-a-judge can accurately extract Schwartz values from opinions, and (3) heuristic proxies can approximate theoretical JS/KL divergences. Assumption 3 is not sufficiently justified in the main text and represents a significant conceptual leap.
- **Are there logical gaps in the derivation or reasoning?** Yes. Section 3.2 derives a rigorous information-theoretic objective (Eq. 1) involving GJS divergence and KL terms. However, the practical implementation (Appendix C.3) replaces probabilistic divergences with set-based metrics (Jaccard diversity) and semantic similarity (1-BERTScore) to handle black-box LLMs. While discussed in Appendix I, the main text does not explicitly state that the optimization is a *heuristic proxy search* rather than exact EM. This creates a disconnect between the theoretical convergence guarantees of Information Maximization and the actual greedy heuristic used.
- **Are there edge cases or failure modes not discussed?** The framework could suffer from reward stagnation or degenerate outputs if the scoring LLMs (P1/P2) share identical alignment filters or cultural biases. Appendix E.5 shows low vs. high scoring questions, but the paper lacks a systematic analysis of how the bandit performs when P1/P2 models are highly aligned, which could produce superficially "controversial" but semantically vacuous questions.
- **For theoretical claims: are proofs correct and complete?** The IM derivation in Appendix D is mathematically sound for a setting with accessible true probability distributions. However, because the discrete, black-box LLM setting forces heuristic approximations, the theoretical convergence guarantees do not directly apply to the implementation used in practice. This should be explicitly clarified to avoid misleading readers.

### Experiments & Results
- **Do the experiments actually test the paper's claims?** Yes. The experiments assess question quality, validity (via value priming), reliability, extensibility (regional/temporal), and distinguishability against baselines, directly targeting the informativeness challenge.
- **Are baselines appropriate and fairly compared?** SVS, ValueBench, and ValueDCG are appropriate static baselines. The comparison is fair, but the evaluation metric relies on the same LLM-as-a-judge paradigm used for optimization, raising potential confounding effects.
- **Are there missing ablations that would materially change conclusions?** Crucially missing is an ablation on the *value extraction model*. The paper uses strong commercial LLMs in P2 for scoring during question generation and then uses a similar model as $p_\omega$ for extracting values during evaluation. This creates a circularity: AdAEM may optimize for questions that trigger the *evaluator's specific notion* of value diversity rather than ground-truth human value distinctions. Testing with an independent, non-overlapping extractor is necessary.
- **Are error bars / statistical significance reported?** Statistical significance is reported for the validity/priming experiments (p-values, Cohen's d) and reliability (Cronbach's $\alpha$). However, the main distinguishability results (Fig. 8 radar plots) lack confidence intervals or statistical testing across the 12,310 questions. Given the noisy nature of LLM sampling, reporting variance across question subsets is essential for ICLR standards.
- **Do the results support the claims made, or are they cherry-picked?** The results generally support the claim that AdAEM yields more distinguishable outputs. However, the priming experiments primarily demonstrate *controllability* (the model can follow a prompt to emphasize a requested value), not that AdAEM accurately measures *inherent* pre-aligned value differences. The paper somewhat conflates priming sensitivity with construct validity.
- **Are datasets and evaluation metrics appropriate?** Using TrueSkill for relative ranking is appropriate for comparative evaluation. The 12k dataset size is substantial, but the reliance on binary LLM-extracted values ($v_i \in \{0,1\}$) discards intensity/confidence information, which may oversimplify nuanced value trade-offs.

### Writing & Clarity
- **Are there sections that are confusing or poorly explained?** The transition from Section 3.2 (theoretical EM with exact probabilities) to Appendix C.3 (heuristic scoring with Jaccard/BERTScore) is jarring. Readers will reasonably expect the implemented objective to match Eq. 1. The paper should explicitly frame the optimization as a *practical, LLM-driven heuristic search* motivated by the theoretical bound, rather than presenting them as functionally equivalent.
- **Are figures and tables clear and informative?** Figures 1, 7, and 8 are clear and effectively communicate the core message. Table 1 and Table 15 provide useful quantitative comparisons. Figure 5's regional distribution and Figure 16's temporal analysis are insightful and well-integrated.

### Limitations & Broader Impact
- **Do the authors acknowledge the key limitations?** Yes. They acknowledge reliance on Schwartz theory (though MFT is explored in Appendix J), English-only scope, assumption simplifications, and potential malicious use of controversial prompts. The ethics statement also details robust filtering safeguards.
- **Are there fundamental limitations they missed?** The paper does not fully address *construct validity*: do LLM responses actually map to human psychological value constructs, or are they learning to mimic discourse patterns associated with Schwartz dimensions? Additionally, the optimization heavily prioritizes *controversy* to maximize distinguishability. This may inadvertently amplify polarizing or extreme framings that don't reflect nuanced human value trade-offs.
- **Are there failure modes or negative societal impacts not discussed?** The safeguards (Llama-Guard) address direct harm, but the paper misses the risk of *benchmark gaming*. Since the framework is automated and open-sourced, model developers could fine-tune on AdAEM-Bench to artificially inflate distinguishability scores, recreating Goodhart's Law. The authors note regeneration is possible but could propose stricter continuous evaluation protocols or cryptographic commitment to question sets.

### Overall Assessment
AdAEM addresses a genuinely important and timely problem in LLM evaluation: overcoming the saturation and contamination of static value benchmarks through a dynamic, self-extensible generation pipeline. The conceptual integration of an information-theoretic objective with a Multi-Armed Bandit for exploration is innovative, and the empirical analysis convincingly demonstrates improved question diversity and model distinguishability compared to static baselines. However, for ICLR standards, the paper requires significant methodological tightening. The most critical concern is the disconnect between the rigorous theoretical derivation (Eq. 1) and the heuristic LLM-driven proxies (Jaccard, BERTScore) used in practice, which undermines the claimed information-maximization guarantees. Additionally, the heavy reliance on LLM-as-a-judge for both optimization and evaluation introduces circularity that risks measuring stylistic divergence rather than true value differences. To strengthen the contribution, the authors must (1) explicitly frame the optimization as a heuristic proxy search in the main text, (2) provide ablations decoupling the question-generation scorer from the value-extraction evaluator to rule out circularity bias, and (3) add statistical variance estimates to the main distinguishability results. With these clarifications and additional controls, this work would make a strong, publishable contribution to dynamic LLM evaluation.

# Neutral Reviewer
## Balanced Review

### Summary
This paper introduces AdAEM, a dynamic, self-extensible evaluation framework designed to address the "informativeness challenge" in LLM value assessment, where static benchmarks yield saturated or contaminated results. The method formulates an information-theoretic objective to maximize value distinguishability and disentanglement across models, optimized via an EM-like alternating process guided by a multi-armed bandit exploration strategy. Extensive experiments across Schwartz’s Value Theory and Moral Foundations Theory demonstrate that AdAEM generates highly informative, regionally/temporally diverse questions that produce more discriminative and reliable value comparisons than existing static benchmarks.

### Strengths
1. **Principled Theoretical Formulation:** The paper clearly formalizes the informativeness challenge and derives a rigorous optimization objective (Eq. 1) balancing Jensen-Shannon divergence (distinguishability) and a disentanglement term. The EM-like alternating optimization (Eqs. 2 & 3) is well-motivated, with clear interpretations for each term (value conformity, semantic diversity, context coherence), bridging information theory and practical LLM prompting.
2. **Comprehensive Empirical Validation:** The evaluation is multi-faceted and meets ICLR's empirical rigor standards. The authors demonstrate *validity* via controlled value priming showing expected psychological shifts (Tables 10, 12), *reliability* via high internal consistency (Cronbach’s α ≈ 0.90), and *question quality* via human evaluation showing +52% gains in controversy and value elicitation (Table 9). Cross-theory instantiation with Moral Foundations Theory (Appendix J) further proves generalizability.
3. **Robust Practical Approximations & Verification:** Acknowledging black-box LLM constraints, the authors replace intractable probability terms with implementable heuristics (e.g., BERTScore, Jaccard). Crucially, Appendix I provides an empirical verification showing these approximations yield highly correlated results (Pearson r=0.856, α=0.898) with exact mathematical implementations, justifying their practical use without sacrificing construct validity.
4. **Effective Mitigation of Contamination & Extensibility:** By using culturally and temporally diverse proxy models (P1/P2) within a Multi-Armed Bandit framework, AdAEM successfully surfaces region-specific and knowledge-cutoff-aligned questions (Figs. 5 & 6). The self-extending design explicitly addresses data leakage concerns that plague static benchmarks, providing a forward-looking evaluation paradigm.

### Weaknesses
1. **Heavy Reliance on LLM-as-a-Judge for Value Annotation:** The entire optimization loop and final scoring depend on GPT-4o/Mini to classify values from model opinions (Listing 4). Although Table 4 reports ~87–93% accuracy, unmeasured judge biases (e.g., preference for certain rhetorical styles, length, or specific value framings) could systematically distort the informativeness score and propagate into the benchmark. The paper lacks a thorough ablation or mitigation strategy for judge-induced noise in the reward signal.
2. **Lack of Comparison to Other Dynamic Generation Methods:** Baselines are restricted to static benchmarks (SVS, ValueBench, ValueDCG). The field has recently explored adaptive or synthetic evaluation frameworks (e.g., DyVal, S-Eval, model-written evaluations). Without comparing against alternative dynamic generation pipelines, it is difficult to isolate how much performance stems from AdAEM's specific information-maximization objective versus the general benefits of iterative question generation.
3. **Computational Cost and Scalability Opaqueness:** The pipeline requires iterative API calls to multiple strong models for exploration, refinement, and scoring across 1,500 budget steps. While a budget parameter *B* controls iterations, the paper does not quantify token/API costs, wall-clock time, or latency. This omission makes it difficult for practitioners to assess the trade-off between benchmark quality and deployment feasibility.
4. **Theoretical-Practical Alignment Gap:** While empirically validated, the substitution of probabilistic terms in Eq. 1/2 with heuristic metrics breaks the strict information-theoretic guarantees of the original objective. The paper acknowledges this but does not provide theoretical bounds on approximation error or discuss conditions under which the heuristic reward might diverge from the true information-maximizing direction (e.g., in cases of semantic saturation or model collapse).

### Novelty & Significance
**Novelty:** The methodological components (EM optimization, multi-armed bandits, proxy model prompting) are individually established in ML and prompt engineering. However, their novel integration into an *information-maximizing, self-extensible value benchmark generator* is a strong conceptual contribution. The formalization of the "informativeness challenge" and the design of a disentanglement-aware reward function are innovative and well-tailored to LLM alignment research.
**Clarity:** The paper is exceptionally well-structured. The mathematical derivations, algorithmic pseudocode (Algorithm 1/2), and comprehensive appendices (prompts, hyperparameters, robustness checks, failure cases) provide a transparent view of the framework. The writing is clear, and visualizations effectively support the narrative.
**Reproducibility:** Strong. The authors provide exhaustive implementation details, exact prompts (Listings 3–8), hyperparameter tables, and promise open-source code/data. The theoretical approximations are empirically verified, and robustness to model participants, question counts, and random splits is thoroughly documented. Minor dependency on commercial API access remains the sole reproducibility constraint.
**Significance:** High. Benchmark saturation and contamination are critical bottlenecks for AI safety and value alignment research. AdAEM’s ability to continuously surface uncontaminated, culturally grounded, and highly discriminative test cases provides a scalable path for tracking LLM alignment dynamics. The framework’s relative-ranking aggregation (TrueSkill) also offers a practical alternative to saturated absolute scoring.

### Suggestions for Improvement
1. **Add a Dynamic Benchmark Baseline:** Compare AdAEM against at least one recent adaptive evaluation method (e.g., DyVal, S-Eval, or iterative model-written evaluations). This would isolate the specific contribution of the information-maximization objective and bandit-driven exploration from general dynamic generation benefits.
2. **Quantify Computational Overhead:** Provide explicit metrics on API token usage, wall-clock generation time, and query counts across different budgets (e.g., B=500 vs. B=1500). Include a cost-benefit analysis to guide practitioners on minimal viable configurations for real-world deployment.
3. **Strengthen Judge Bias Analysis:** Investigate how LLM-as-a-judge inaccuracies impact the optimization loop. For example, run a small experiment using multiple diverse judge models or apply position/length debiasing during value classification. Discuss how noisy reward signals might affect the UCB selection in Algorithm 1 and whether smoothing or confidence-weighted rewards could improve stability.
4. **Formalize Approximation Mapping & Limitations:** In Section 3.2 or Appendix I, explicitly map each heuristic (BERTScore, Jaccard, GPT-4o probability) to its corresponding theoretical term in Eq. 2/3. Briefly discuss the conditions under which these proxies fail (e.g., high semantic redundancy, nuanced value trade-offs) and propose a sensitivity analysis on key approximation thresholds (e.g., similarity threshold ε).

# Spark Finder Review
## How to Improve This Paper

### Missing Experiments (top 3-5 only)
1. **Ablation of the optimization objective components.** Remove the disentanglement and distinguishability terms individually to prove they are strictly necessary for generating value-evoking questions, rather than the observed quality simply emerging from basic semantic diversification or prompt length increases.
2. **Empirical data contamination verification.** Add a formal contamination check (e.g., querying pre-/post-cutoff model versions or running explicit leakage detectors on the generated 12k questions) because the central claim of mitigating memorization is currently unsupported by evidence.
3. **Comparison to state-of-the-art dynamic/adversarial test generation baselines.** Benchmarking only against static psychometric surveys (SVS, ValueBench) ignores recent LLM-driven dynamic evaluation works, making the claim of superior "informativeness" ungrounded against the actual methodological baseline for adaptive testing.

### Deeper Analysis Needed (top 3-5 only)
1. **Validation beyond explicit value priming.** The validity analysis relies on prompting models to adopt a value via system instructions, which only proves the pipeline can detect *instructed* behavior, not that it captures *latent, inherent* value orientations during standard querying.
2. **Disentanglement of value differences from sampling variance and instruction-following styles.** The optimization may simply exploit temperature-induced randomness or stylistic divergence rather than genuine value disagreements; controlled experiments holding generation parameters constant are required to isolate semantic value shifts.
3. **Correlation of AdAEM scores with concrete downstream model behaviors.** To substantiate the claim that revealed value differences diagnose "misalignment" or "cultural bias," demonstrate that AdAEM rankings predict actual risk outcomes (e.g., safety violations, biased translations) rather than just differing on synthetic probes.

### Visualizations & Case Studies
1. **Expert-annotated side-by-side comparisons of generated vs. baseline questions with full response traces.** Show human-labeled value scores alongside model outputs to verify that the "value differences" are psychologically meaningful trade-offs rather than superficial semantic disagreements or framing artifacts.
2. **Optimization trajectory mapping question semantics and value scores across iterations.** Without visualizing how questions evolve and how the distinguishability score converges, the claim of "probing value boundaries" remains abstract and unverifiable against simple heuristic drift.
3. **Case studies of failure modes or degenerate convergence.** Provide examples where the algorithm optimizes toward irrelevant, trivially agreed-upon, or nonsensical questions despite high heuristic scores, which would expose the limitations of the practical approximations used in place of Eq.(1).

### Obvious Next Steps
1. **Full reconciliation of the theoretical EM derivation (Eq. 1) with the practical heuristic implementation.** The paper claims a rigorous information-theoretic foundation but replaces probability terms with Jaccard/BERTScore proxies in practice; a direct empirical bridge or formal justification is required to maintain theoretical credibility.
2. **Ground-truth alignment with established human cross-cultural value surveys.** Validate AdAEM's regional/cultural rankings against real-world human data (e.g., World Values Survey) to prove the framework captures actual cultural value differences rather than training corpus artifacts or LLM stereotype mimicry.
3. **Compute and API cost analysis relative to static benchmarks.** Report the financial cost, latency, and compute overhead for generating and scoring the 12k questions to demonstrate the framework's practical viability, as ICLR reviewers require evidence that a proposed dynamic method is scalable without prohibitive resource demands.

# Final Consolidated Review
## Summary
This paper introduces AdAEM, a dynamic, self-extensible evaluation framework designed to overcome the saturation and data contamination of static LLM value benchmarks. It formalizes value elicitation as an information-theoretic optimization problem, solved via an EM-like alternating process combined with a Multi-Armed Bandit exploration strategy. The method iteratively generates and refines controversial, regionally/temporally grounded questions to maximize distinguishability and disentanglement across diverse models, yielding a large-scale benchmark with comprehensive empirical validation.

## Strengths
- **Addresses a critical, timely bottleneck:** The "informativeness challenge" in value evaluation is clearly motivated and grounded in measurement theory. Shifting from static recall to active boundary-probing directly tackles benchmark saturation and leakage.
- **Comprehensive empirical validation:** The evaluation meets high empirical standards, demonstrating validity via controlled value priming (significant same-group/opposite-group shifts), reliability via strong internal consistency (Cronbach's α ≈ 0.90), and question quality via human evaluation (+52% gains in controversy and value elicitation).
- **Strong generalization across theories:** Successful instantiation with both Schwartz's Theory of Basic Values and Moral Foundations Theory proves the framework is not tied to a single psychological construct, and the TrueSkill aggregation provides robust comparative rankings.

## Weaknesses
- **Unquantified circularity and scorer bias in LLM-as-a-Judge:** Both the optimization reward loop and the final value extraction heavily rely on proprietary commercial LLM APIs (e.g., GPT-4o/Mini). Because the same class of models judges "informativeness" during generation and classifies values during evaluation, it is unclear whether AdAEM discovers genuine psychological value boundaries or merely exploits the scorer's specific alignment filters, stylistic preferences, or prompt-following idiosyncrasies. Without decoupling the optimizer judge from the evaluator or testing structural robustness across disparate judge types, the measured "value differences" risk reflecting proxy model artifacts rather than target model traits.
- **Significant theory-practice disconnect obscures guarantees:** The main text derives a rigorous Information Maximization objective with probabilistic guarantees (Eq. 1), but the actual black-box implementation replaces intractable probability terms with set-based heuristics (Jaccard, BERTScore). While Appendix I provides empirical correlation scores (r=0.856) to justify this substitution, the core presentation overstates theoretical convergence and leaves readers without clear bounds on when the heuristics diverge from the true information-maximizing direction (e.g., under semantic saturation or model collapse).
- **Validation is artificially constrained and lacks dynamic baselines:** Construct validity rests almost entirely on explicit value priming, which demonstrates controllability/instructability rather than the measurement of *inherent* pre-aligned value orientations. Furthermore, the paper only compares against static psychometric surveys. Without benchmarking against recent dynamic/adaptive evaluation pipelines (e.g., DyVal, S-Eval, iterative model-written evaluations), it is impossible to isolate how much performance stems from AdAEM's specific information-maximization objective versus the general benefits of iterative question generation. Correlations with human cross-cultural surveys or downstream safety behaviors are also absent, weakening claims of diagnosing real-world misalignment.

## Nice-to-Haves
- Provide explicit API/token costs, query counts, and wall-clock latency across different budgets to assess practical deployment feasibility.
- Include confidence intervals or variance estimates on the distinguishability results (Fig. 8) to account for LLM sampling stochasticity.
- Run a controlled experiment isolating value shifts from generation temperature/styling artifacts, and ablate individual objective terms (disentanglement vs. distinguishability) to prove their strict necessity.
- Conduct an explicit empirical leakage test on the generated 12k questions using detection tools or pre-/post-cutoff model queries to concretely verify the contamination mitigation claim.

## Novel Insights
Reframing LLM value assessment as an active boundary-probing problem rather than passive static querying is a conceptually strong paradigm shift. The integration of information maximization with multi-armed bandit exploration creates a self-extending evaluation loop that inherently adapts to model advancements, moving the field away from brittle, memorization-prone benchmarks toward dynamic, co-evolving diagnostic tools that surface culturally and temporally specific friction points.

## Potentially Missed Related Work
- None identified. The paper adequately surveys value benchmarking and dynamic evaluation literature.

## Suggestions
- Decouple the optimization reward model from the final value extractor. Run a targeted ablation using structurally different judge models (e.g., open-weight classifiers, human-annotated subsets, or adversarial prompters) to rigorously rule out scorer-specific bias and confirm that generated questions generalize across evaluators.
- Revise Section 3.2 to explicitly frame the pipeline as a heuristic proxy search motivated by theoretical bounds, and formally map each practical metric (BERTScore, Jaccard, GPT probability approximations) to its corresponding probabilistic term in Eq. 1/2. Clarify the conditions under which the heuristic reward may fail.
- Introduce at least one recent dynamic/adaptive evaluation baseline to isolate the contribution of the information-maximization objective. Additionally, add a validation experiment that correlates AdAEM's regional/cultural rankings with established human cross-cultural survey data (e.g., World Values Survey) or concrete downstream alignment failure modes to ground the psychological claims in observable reality.
